# Eclipse Help Generation

## Overview

The Xtext Hugo build automatically generates Eclipse help format output alongside the regular website. The Eclipse help files are placed in `xtext-website/public/eclipse/`.

## Output Format

Eclipse help pages use a simplified HTML structure:

```html
<html>
<head>
  <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
  <title>Page Title</title>
  <link href="book.css" rel="stylesheet" type="text/css">
  <link href="code.css" rel="stylesheet" type="text/css">
  <link rel="home" href="index.html" title="">
</head>
<body>
  <!-- Content -->
</body>
</html>
```

## Copying to Eclipse Xtext Repository

The Eclipse help files need to be copied to the main Xtext repository for inclusion in the Eclipse help system.

### Target Location

```
~/xtext-main/git/xtext/org.eclipse.xtext.doc/contents/
```

### Manual Copy Process

```bash
# From the xtext-website repository
cd xtext-website

# Build the site (generates both web and eclipse output)
hugo

# Copy Eclipse help to Xtext repo
cp -r public/eclipse/* ~/xtext-main/git/xtext/org.eclipse.xtext.doc/contents/
```

### Automating with a Script

Create a script `copy-eclipse-help.sh`:

```bash
#!/bin/bash

# Configuration - adjust these paths to your setup
XTEXT_WEBSITE_DIR="${XTEXT_WEBSITE_DIR:-$(pwd)}"
XTEXT_REPO_DIR="${XTEXT_REPO_DIR:-$HOME/xtext-main/git/xtext}"
XTEXT_DOC_DIR="$XTEXT_REPO_DIR/org.eclipse.xtext.doc/contents"

echo "Building Hugo site..."
cd "$XTEXT_WEBSITE_DIR/xtext-website"
hugo

echo "Copying Eclipse help files to $XTEXT_DOC_DIR..."
mkdir -p "$XTEXT_DOC_DIR"
cp -r public/eclipse/* "$XTEXT_DOC_DIR/"

echo "Done!"
echo "Files copied to: $XTEXT_DOC_DIR"
```

Usage:
```bash
export XTEXT_REPO_DIR=~/path/to/xtext/repo
./copy-eclipse-help.sh
```

## toc.xml Generation

The Eclipse help system requires a `toc.xml` file for navigation. This file lists all documentation pages and their structure.

### Current Approach

The `toc.xml` is generated by a Java tool (`gen_toc.jar`) in the Xtext repository. After copying the HTML files:

```bash
cd ~/xtext-main/git/xtext/org.eclipse.xtext.doc
# Run the TOC generator (adjust command based on existing build setup)
java -jar tools/gen_toc.jar
```

### toc.xml Structure

The file should look like:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<toc label="Xtext Documentation">
  <topic label="Getting Started" href="documentation/301_grammarlanguage.html">
    <topic label="Grammar Language" href="documentation/301_grammarlanguage.html"/>
    <topic label="Ten Minutes Tutorial" href="documentation/102_domainmodelwalkthrough.html"/>
    ...
  </topic>
  ...
</toc>
```

## CSS Files

Eclipse help requires two CSS files that should be present in the output:

- `book.css` - Main styling for Eclipse help
- `code.css` - Code highlighting styles

These are typically provided by the Eclipse help system or copied from:

```
org.eclipse.xtext.doc/contents/book.css
org.eclipse.xtext.doc/contents/code.css
```

## Build Integration

For automated builds, add a step to the Xtext repository's build:

```groovy
// In Xtext repo's Jenkinsfile or build script
stage('Update Documentation') {
    steps {
        sh '''
            # Build and copy from xtext-website
            cd /path/to/xtext-website
            hugo
            
            # Copy to Xtext doc folder
            cp -r public/eclipse/* /path/to/xtext/org.eclipse.xtext.doc/contents/
            
            # Generate toc.xml
            cd /path/to/xtext/org.eclipse.xtext.doc
            # ... run TOC generation ...
        '''
    }
}
```

## Verification

To verify the Eclipse help output:

1. Check that HTML files are generated:
   ```bash
   ls xtext-website/public/eclipse/documentation/
   ```

2. Verify the HTML structure:
   ```bash
   head -20 xtext-website/public/eclipse/documentation/301_grammarlanguage.html
   ```

3. In Eclipse, open the help system and navigate to Xtext documentation to verify rendering.

## Differences from Website Version

The Eclipse help version differs from the website:

| Feature | Website | Eclipse Help |
|---------|---------|--------------|
| Layout | Full page with header/footer | Content only |
| CSS | Bootstrap + custom | book.css + code.css |
| Navigation | Website menus | Eclipse help TOC |
| Base URLs | `https://eclipse.dev/Xtext/` | Relative paths |
